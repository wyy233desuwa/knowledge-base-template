# 项目知识库

这是一个面向 AI Coding / Agent 协作的项目知识库模板。

它的目标不是替代项目文档，而是帮助 AI 在新对话、新任务、新需求中快速理解项目上下文，沉淀真实执行经验，避免每次都从零开始猜测项目规则。

## 适合什么场景

- 你经常用 AI 协助开发、排查 bug、写方案或复盘需求。
- 项目中有很多“只靠口头记忆”的实现习惯、历史坑、验证命令和技术口径。
- 你希望每次需求都能沉淀为可复用的案例，而不是散落在聊天记录里。
- 你希望后续 AI 能先读项目知识库，再结合当前代码做判断。

## 核心设计

本模板采用 Markdown + 中文 frontmatter 的方式组织知识。每个文档都包含统一中文元数据：

```md
---
文档类型: "案例记录"
当前状态: "计划记录"
主题: "需求名称"
创建日期: "YYYY-MM-DD"
最近整理日期: "YYYY-MM-DD"
标签:
  - "案例"
  - "项目经验"
来源:
  - "人工整理"
关联文档: []
当前有效性: "以当前项目代码、当前结论和案例记录索引为准"
---
```

字段统一使用中文，方便团队成员和 AI 同时阅读。

## 每个文件的作用

| 文件 | 作用 |
| --- | --- |
| `README.md` | 知识库入口，说明项目知识库定位和使用原则 |
| `AI新对话启动协议.md` | 规定 AI 每次接手任务时优先读取哪些文档 |
| `知识库索引.md` | 二级入口，不直接堆所有历史文档 |
| `知识库使用说明.md` | 说明如何新增和维护案例记录 |
| `知识库维护日志.md` | 记录知识库字段、结构和维护规则的变化 |
| `当前结论.md` | 只记录当前有效口径、已知风险和最新结论 |
| `实施决策记录.md` | 作为可复用规则入口，沉淀跨案例适用的实施口径、字段口径、验证约束和历史误判 |
| `案例记录索引.md` | 按需求索引所有案例 |
| `历史文档目录.md` | 追溯原始方案、复验结论和实施总结 |
| `历史节点-计划.md` | 按计划阶段追溯历史方案 |
| `历史节点-建议.md` | 按建议阶段追溯复查问题和风险 |
| `Bug处理流程.md` | 规定 bug、冲突、异常行为的处理流程 |
| `CASE-YYYYMMDD-需求名-001.md` | 单个需求的案例模板 |

## 案例闭环

每个需求建议沉淀为一条 CASE 文档，固定包含五个部分：

- 计划：当时准备怎么做、范围是什么、参考对象是什么。
- 执行：实际改了哪些文件、走了哪些步骤。
- 建议：复查时发现的问题、风险和优化方向。
- 修改：根据建议实际修了什么、哪些没修、为什么。
- 总结：最终结论、可复用规则、下次 AI 应该优先记住什么。

## 如何用于新项目

1. 复制整个模板目录到你的项目知识库目录。
2. 修改本文件中的项目名称、代码路径、知识库路径、技术栈。
3. 修改 [AI新对话启动协议.md](./AI新对话启动协议.md) 中的项目信息。
4. 在 [当前结论.md](./当前结论.md) 中写入当前项目真实有效的技术口径。
5. 在 [实施决策记录.md](./实施决策记录.md) 中写入项目约束、技术决策和历史误判。
6. 每次新需求复制 [CASE-YYYYMMDD-需求名-001.md](./CASE-YYYYMMDD-需求名-001.md) 新建案例。
7. 完成需求后同步更新 [案例记录索引.md](./案例记录索引.md) 和必要的当前结论。

## 知识库原则

- 一次需求形成一条案例记录。
- 案例记录必须保留当时的判断过程，不能只写最终结论。
- 历史误判不要删除，要在总结里标注“后来证明不成立”。
- 索引不直接堆所有原始文档；原始文档从 [历史文档目录](./历史文档目录.md) 追溯。
- 跨案例反复适用的规则统一沉淀到 [实施决策记录](./实施决策记录.md)，避免规则散落。
- 每个 Markdown 文档开头保留中文 frontmatter，字段统一使用中文。
- 涉及测试时，必须记录命令和限制。
- 新对话启动时按 [AI新对话启动协议](./AI新对话启动协议.md) 读取知识库，不要默认展开全部历史文档。
