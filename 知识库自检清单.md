# 知识库自检清单

这个清单用于每次新增、收口或重整知识库后快速检查，避免案例写了但入口、总口径或可复用规则没同步。

## 1. 文件结构

- [ ] 新增 Markdown 文件都有中文 frontmatter。
- [ ] frontmatter 字段齐全：`文档类型`、`当前状态`、`主题`、`创建日期`、`最近整理日期`、`标签`、`来源`、`关联文档`、`当前有效性`。
- [ ] 文件名符合 CASE 命名规则。
- [ ] Markdown 相对链接没有断链。

## 2. 案例入口

- [ ] 新 CASE 已加入 [案例记录索引.md](./案例记录索引.md)。
- [ ] 同一 CASE 的计划和执行总结优先合并为一行入口。
- [ ] 被新版本吸收的旧计划仍保留入口，但状态写清“已被 XXX 吸收/追溯使用”。
- [ ] 重要修改意见已进入“关联修改意见文档”或对应案例入口。

## 3. 总口径同步

- [ ] 当前仍有效的业务结论已同步到 [当前结论.md](./当前结论.md)。
- [ ] 跨案例可复用的规则已同步到 [实施决策记录.md](./实施决策记录.md)。
- [ ] 知识库结构或维护规则变化已写入 [知识库维护日志.md](./知识库维护日志.md)。
- [ ] 需要按阶段追溯的计划和建议已同步到 [历史节点-计划.md](./历史节点-计划.md) 或 [历史节点-建议.md](./历史节点-建议.md)。

## 4. 执行与验证

- [ ] 执行总结写清实际修改文件。
- [ ] 验证命令写清执行目录、命令和结果。
- [ ] 不能执行的验证写清原因和剩余风险。
- [ ] 用户已明确验收的内容，状态和正文保持一致。

## 5. 质量提醒

- [ ] 没有把历史方案当成当前事实。
- [ ] 没有把计划里的过时代码示例标成最终口径。
- [ ] 没有覆盖原始计划、原始建议和执行总结。
- [ ] 当前代码事实、用户最新要求、当前结论之间没有明显冲突。
