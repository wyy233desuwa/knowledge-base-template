# AI 新对话启动协议

这个文档规定 Agent 每次开始新对话时如何使用本项目知识库。目标是快速继承项目上下文，同时避免被历史文档带偏。

## 2. 必读启动文档

每次新对话开始，先快速读取以下 4 个文档：

| 顺序 | 文档 | 读取目的 |
| --- | --- | --- |
| 1 | [README.md](./README.md) | 理解知识库定位和整体维护原则 |
| 2 | [知识库索引.md](./知识库索引.md) | 获取二级整理文档入口和当前案例列表 |
| 3 | [当前结论.md](./当前结论.md) | 获取当前有效状态、已知风险和最新口径 |
| 4 | [实施决策记录.md](./实施决策记录.md) | 获取跨案例复用规则、历史误判和执行约束 |

这一步只做快速阅读，不需要展开所有历史资料。

Windows PowerShell 读取中文 Markdown 时，统一显式使用 UTF-8，例如：

```powershell
Get-Content -LiteralPath "知识库路径\README.md" -Encoding UTF8 -Raw
```

不要把未指定编码导致的乱码误判为文档内容损坏。

## 3. 元数据读取规则

每个 Markdown 文档开头都应保留中文 frontmatter。Agent 读取文档时，先看 frontmatter 判断文档用途和有效性，再阅读正文。

| 字段 | 用途 |
| --- | --- |
| `文档类型` | 判断是入口说明、启动协议、案例记录、索引、当前结论、决策记录、历史目录、历史节点、流程规范还是维护日志 |
| `当前状态` | 判断文档当前处于当前有效、追溯使用、计划记录、复查意见、待批准、已完成、案例总结等状态 |
| `主题` | 文档主题或 CASE 名称 |
| `创建日期` | 文档或 CASE 对应日期 |
| `最近整理日期` | 最近一次知识库结构化整理日期 |
| `标签` | 便于按主题快速筛选 |
| `来源` | 文档来源或整理来源 |
| `关联文档` | 关键关联入口，不要求穷举所有正文链接 |
| `当前有效性` | 判断该文档与当前代码、当前结论、案例索引之间的优先级关系 |

字段名统一使用中文，不新增英文同义字段。

## 4. 条件读取文档

| 触发条件 | 继续读取 |
| --- | --- |
| 任务涉及历史方案、历史建议、复查或相似需求 | [案例记录索引.md](./案例记录索引.md)、对应 CASE 文档 |
| 任务需要追溯原始方案来源 | [历史文档目录.md](./历史文档目录.md) |
| 任务只想看最初计划 | [历史节点-计划.md](./历史节点-计划.md) |
| 任务只想看复查建议 | [历史节点-建议.md](./历史节点-建议.md) |
| 任务涉及新增案例或整理知识库 | [知识库使用说明.md](./知识库使用说明.md)、[知识库维护日志.md](./知识库维护日志.md) |
| 任务涉及 bug、冲突、异常行为或用户指出“不对” | [Bug处理流程.md](./Bug处理流程.md) |

## 5. 优先级规则

Agent 判断冲突时按以下优先级处理：

1. 用户当前对话里的最新要求。
2. 当前代码和实际文件内容。
3. [当前结论.md](./当前结论.md)。
4. [实施决策记录.md](./实施决策记录.md)。
5. CASE 案例文档。
6. 历史节点文档。
7. 原始历史文档。

原始历史文档只用于追溯，不作为当前最终口径。

## 6. CASE 命名格式

```text
CASE-YYYYMMDD-计划名-三位序号.md
CASE-YYYYMMDD-计划名-三位序号-修改意见.md
CASE-YYYYMMDD-计划名-三位序号-Bug处理计划.md
```

三位序号是同一计划名下的版本号，不是当天全局编号。

## 7. 禁止事项

- 不要每次新对话都完整读取所有历史文档。
- 不要把历史方案里的旧判断直接当成当前事实。
- 不要只读知识库就跳过当前代码检查。
- 不要在用户明确要求保持不变时反复建议改动。
- 遇到 bug、冲突或异常行为时，不要直接改代码；先说明原因，写处理计划，等待用户批准。
